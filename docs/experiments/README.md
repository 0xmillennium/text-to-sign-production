# Experiment Records

Future experiments should create one Markdown record per meaningful run or comparison.

## Recording Guidance

- Use the experiment template as a starting point.
- Prefer one log per command or tightly related run group.
- Record exact dataset references, config identifiers, and commands.
- Summarize outcomes clearly enough that another researcher can understand what happened.

## Sprint 1 Boundary

Sprint 1 does not ship real experiments. This directory exists so later work can adopt a
consistent documentation habit from the start.
